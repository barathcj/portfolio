TANGO- A Dance of the Web

Date: Sep 2013
Worked with Javascript, HTML, CSS

Role: Front end developer
Team mates: Ramya Mallya, Ayo Olubeko, Andre Le


WHAT IS IT?

Tango is a prototype for a news website created for HackCMU. It is adaptive to the distance the user is from the screen and presents the news in 3 different modes. The user sees a detailed mode when he is close to the screen. If he leans back on his chair, he can browse through the various articles in the browse mode. If he leaves his chair and is away from the screen, the website plays the news as a video. It uses a web camera to estimate the distance of the user's face.



WHY WAS IT MADE?

Tango was made as a project for HackCMU 2013. It was made in one day and went on to win the most innovative project award!



HOW WAS IT DONE?

Tango uses a face tracking javascript library called. Through this, we were able to measure the distance of the user's face from the screen. We then calliberated threshold distances, and adjusted the layout of the page based on this

TICKOLO - An Android App for Fan to Fan Ticket Exchange

Date: Nov 2013

Role: Software, UI Design

WHAT IS IT?

Tickolo is a prototype for a ticket exchange platform in which users can buy and sell spare tickets at less than face value. It was made as an assignment for the course Software Structures for User Interfaces. This app currently works on the android platform and a beta version will be launched by the end of May 2014.

Bicycle Air Monitoring Project

Date: March 2014
Worked with Axure RP, Adobe Illustrator, inDesign

Role: UX designer
Team mates: Ramya Mallya, Phillip Marchetti, Andre Le


WHAT IS IT?

The Bicycle Air Monitoring Project is run by the Group Against Smog and Pollution(GASP) and measures air particulate matter count in Pittsburgh. Air monitors are mounted on bikes and volunteers collect data on their daily routes. This data is then represented on a map to provide insights on the level of pollution in Pittsburgh.



HOW DID WE HELP?

GASP approached us wanting a more user-centred approach to the Bicycle Air Monitoring Project. Over the course of a semester we determined the target audience and re-designed the map so as to make the biggest impact on the Pittsburgh community. The end result was a more compelling map which facilitated a dialogue and gave accurate recommendations to the residents of Pittsburgh.



HOW WAS IT DONE?

We conducted surveys, interviews and did an extensive literature review to understand who the target audience would be for this map. We then did a Heuristic Evaluation of the existing system and noted down ways that we could improve it. After 5 rounds of iterative design, we delivered a high-fidelity Axure prototype along with design specs to take the project forward.
